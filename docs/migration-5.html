<h1 id="migrating-from-4-x-to-5-x">Migrating from 4.x to 5.x</h1> <ul> <li><a href="/docs/migration-4.html">Migrating from 3.x to 4.x</a></li> <li><a href="/docs/migration-3.html">Migrating from 2.x to 3.x</a></li> <li><a href="/docs/migration-2.html">Migrating from 1.x to 2.x</a></li> <li><a href="/docs/migration.html">Migrating from 0.x to 1.x</a></li> </ul> <h2 id="release-notes-and-breaking-changes">Release notes and breaking changes</h2> <p>With version 5.0.0, <code>router5</code> is now a monorepo: all repos have been imported into <a href="https://github.com/router5/router5">https://github.com/router5/router5</a>. This will make maintenance and release of new versions easier.</p> <h4 id="bug-fixes">Bug fixes</h4> <p>Not found state is now passed to middleware functions on start: if you use <code>allowNotFound</code> option and have custom middleware functions, make sure they still work.</p> <h4 id="default-options">Default options</h4> <ul> <li>router <code>strictQueryParams</code> option is now <code>false</code> by default: if you currently don&#39;t specify that option, you need to explicitely set it to <code>true</code> to keep the same behaviour.</li> <li>browser plugin option <code>preserveHash</code> is now <code>true</code> by default</li> </ul> <h3 id="packages-renamed">Packages renamed</h3> <ul> <li><code>router5.helpers</code> package has been renamed to <code>router5-helpers</code></li> <li><code>router5.transition-path</code> package has been renamed to <code>router5-transition-path</code></li> </ul>
